Source des données

Les données utilisées dans ce projet proviennent de la plateforme Spotify. En effet, Spotify offre la possibilité d'accéder à des données personnelles sur les utilisateurs, via son API ouverte. Ces données incluent des informations détaillées sur les genres musicaux écoutés, les artistes, les morceaux et d'autres comportements d'écoute.

Chaque utilisateur peut obtenir ses propres données via la plateforme, après avoir accordé l'accès à son compte Spotify. Les informations utilisées dans ce projet sont donc personnelles et spécifiques à chaque utilisateur, respectant la confidentialité et les choix de l'utilisateur.

Objectif du projet

L'objectif de ce projet est de permettre aux utilisateurs de visualiser leurs données musicales personnelles, en mettant en lumière les genres musicaux qu'ils écoutent, ainsi que l'évolution de leurs habitudes d'écoute selon différents contextes tels que les périodes de cours ou de travail.

Les visualisations proposées sont interactives, offrant à chaque utilisateur la possibilité d'explorer ses propres préférences musicales et de les comparer avec celles d'autres utilisateurs, permettant ainsi une analyse plus approfondie de ses habitudes d'écoute.

Traitement des données

Les données fournies par Spotify sont initialement stockées dans une base de données locale. Toutefois, ces données ne contiennent pas toutes les informations nécessaires, notamment les genres des titres. Pour enrichir ces données, nous avons utilisé l'API de Spotify, qui nous permet d'obtenir des informations détaillées sur les genres associés à chaque morceau de musique.

Après cette phase d'enrichissement et de traitement, les données ont été agrégées et transformées dans des formats adaptés aux différents types de graphiques et visualisations proposées dans ce projet. Cela permet d'offrir une représentation claire et précise des habitudes d'écoute de chaque utilisateur.

Source des codes utilisés

  • Le projet utilise principalement la bibliothèque JavaScript D3.js pour la création de graphiques interactifs.
  • Le Donut Chart a été inspiré du code fourni sur Observable.
  • La Timeline a été inspirée du projet Patternfly-Timeline dans la fonctionnalité, notamment pour l'utilisation de brush.
  • Le Spider Chart a été inspiré du code fourni sur D3 Spider Chart Tutorial.
